# 227 — AI Prompt Engineering for Compliance

## Objective
Optimize Claude prompts for communication review (task 214) and notice generation (task 215). Design prompts that include jurisdiction-specific context, ensure consistent output formats, and add escalation triggers for high-risk scenarios (eviction, fair housing, ADA).

## Context
Reference: `features/inprogress/P3-003-legal-compliance-engine/README.md`

Prompt quality directly impacts the usefulness of AI-powered compliance checking. Well-designed prompts ensure Claude returns structured, actionable findings that landlords can trust.

## Implementation

1. **Communication Review Prompt** — For POST /api/compliance/review
   - Create `apps/web/lib/compliance/prompts.ts` with prompt templates
   - Prompt structure:
     ```
     # Task
     You are a legal compliance expert for landlord-tenant law.
     Review the following landlord message for compliance issues.

     # Jurisdiction Context
     State: [state_code]
     City: [city]
     Applicable Rules:
     [List of relevant jurisdiction_rules.rule_text]

     # Message to Review
     [landlord_message_text]

     # Instructions
     Identify and report:
     1. Fair housing violations
        - Discriminatory language or criteria
        - Violations of protected class laws
     2. Improper notice language
        - Missing required disclosures
        - Incorrect notice periods
        - Improper formatting for jurisdiction
     3. Liability risks
        - Threats or illegal demands
        - Attempts to circumvent law
        - Missing statutory language

     # Output Format
     Return ONLY valid JSON (no markdown, no explanation):
     {
       "findings": [
         {
           "severity": "warning" | "error",
           "type": "fair_housing" | "notice_language" | "disclosure" | "other",
           "flagged_text": "exact text from message",
           "reason": "explanation",
           "suggestion": "how to correct"
         }
       ],
       "overall_risk_level": "low" | "medium" | "high",
       "safe_to_send": boolean,
       "escalation_required": boolean,
       "escalation_reason": "reason if escalation needed"
     }

     # Escalation Triggers
     If message involves:
     - Eviction or lease termination: Set escalation_required: true
     - Fair housing violation: Set escalation_required: true
     - ADA accommodation denial: Set escalation_required: true
     Then suggest: "Consult attorney before sending"
     ```

2. **Notice Generation Prompt** — For POST /api/compliance/notices/generate
   - Prompt structure:
     ```
     # Task
     You are an expert in landlord-tenant law and document generation.
     Generate a legally-sound notice for a property in [state_code], [city].

     # Jurisdiction Rules and Requirements
     [List jurisdiction_rules.rule_text for relevant topics]
     Key Requirements:
     - Notice period: [X] days
     - Required language: [statutory text]
     - Format: [jurisdiction format requirements]

     # Notice Details
     Notice Type: [entry | lease_violation | rent_increase | eviction]
     Property Address: [address]
     Tenant Name: [name]

     # Context Data
     [context object from request, e.g.]
     Issue: [issue_description]
     Date: [date]
     Amount: [amount, if applicable]

     # Output Format
     Generate a professional notice letter that:
     1. Opens with date and formal salutation
     2. States the purpose clearly
     3. Includes all required statutory citations
     4. Specifies the notice period and effective date
     5. Uses proper legal language for [jurisdiction]
     6. Closes with signature block

     Include a disclaimer: "This notice was generated by an AI tool. It should be reviewed by a licensed attorney before delivery."

     Return ONLY the notice text (no JSON, no explanation).
     ```

3. **Prompt refinement guidelines**:
   - **Context inclusion**: Always include relevant jurisdiction rules
   - **Output format**: Specify exact JSON structure for review, plain text for notices
   - **Escalation triggers**: Include specific scenarios (eviction, fair housing, ADA)
   - **Statute citations**: Ask Claude to cite specific statutes
   - **Tone**: Professional, risk-averse (favor safer interpretations)
   - **Disclaimers**: Always included in generated content
   - **Edge cases**: Specify how to handle ambiguous situations

4. **Testing and iteration**:
   - Create test cases in `apps/web/lib/compliance/prompts.test.ts`
   - Test scenarios:
     - Fair housing violation (discriminatory language)
     - Improper notice period
     - Missing disclosure
     - Eviction scenario (escalation)
     - ADA scenario (escalation)
     - Clean message (no findings)
   - Verify:
     - Output format is valid JSON (for review) or plain text (for notices)
     - Findings are accurate and actionable
     - Escalation flags are appropriate
     - Statute citations are correct

5. **Prompt versioning**:
   - Create `apps/web/lib/compliance/prompts.v1.ts` (initial version)
   - Comment with date and iteration notes
   - Keep old versions for rollback if needed

6. **Jurisdiction-specific variations**:
   - Some prompts may need state-specific language
   - Create helpers to inject jurisdiction-specific context:
     ```typescript
     function buildReviewPrompt(messageText, jurisdiction) {
       const rules = await getJurisdictionRules(jurisdiction);
       const rulesText = rules.map(r => `${r.topic}: ${r.rule_text}`).join('\n');
       return `# Jurisdiction Context\nState: ${jurisdiction.state_code}\n${rulesText}...`;
     }
     ```

7. **Escalation handling**:
   - Define escalation triggers:
     - Eviction/lease termination
     - Fair housing violations
     - ADA accommodation denials
     - Rent control violations
     - Retaliation scenarios
   - When escalation_required: true:
     - Add warning to response: "Attorney review recommended"
     - Log as high-priority in audit trail
     - Optionally queue for manual review

8. **Error recovery**:
   - If Claude returns invalid JSON/format:
     - Log error with request context
     - Return error response to client: "Unable to generate review. Try again."
     - Don't pass malformed data to frontend
   - If Claude seems to miss escalation trigger:
     - Fallback to conservative interpretation: "If unsure, escalate"

9. **Documentation**:
   - Add comment blocks to prompt templates explaining:
     - What jurisdiction context to include
     - Expected output format
     - How escalation triggers work
     - Examples of good/bad messages

10. **Update endpoints.md**
    - Document prompt version used by each endpoint
