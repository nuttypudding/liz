# 215 — Notice Generation API Route

## Objective
Build an API endpoint that uses Claude to generate legally-templated notices customized to the property's jurisdiction. Supports multiple notice types (entry, lease violation, rent increase, eviction) and includes required statutory citations and disclaimers.

## Context
Reference: `features/inprogress/P3-003-legal-compliance-engine/README.md`

Generating compliant notices is a core compliance feature. Claude creates jurisdiction-specific templates that include all required statutory language and notice periods, saving landlords time and reducing legal risk.

## Implementation

1. **POST /api/compliance/notices/generate**
   - Auth: Verify user owns the property
   - Request body:
     ```json
     {
       "property_id": "uuid",
       "notice_type": "entry|lease_violation|rent_increase|eviction",
       "context": {
         "tenant_name": "string",
         "issue_description": "string (e.g., 'Access for annual inspection')",
         "proposed_date": "ISO date (e.g., for entry notices)",
         "rent_increase_amount": "number or string (e.g., '$100' or '10%')",
         "effective_date": "ISO date (for rent increase, lease termination)",
         "additional_details": "string (optional context)"
       }
     }
     ```

2. **Fetch jurisdiction rules**:
   - Query jurisdiction_rules for property's jurisdiction
   - Extract required notice periods, statutory text, and requirements
   - Pass rules to Claude as context

3. **Claude prompt design**:
   - Provide jurisdiction rules and requirements
   - Specify notice type and context data
   - Ask Claude to generate:
     - Professional, legally-sound notice text
     - Include all required statutory citations
     - Include required notice period language
     - Use jurisdiction-specific terminology
     - Format for professional delivery (letter format)
   - Ensure disclaimer is included in generated text

4. **Response format**:
   ```json
   {
     "property_id": "uuid",
     "notice_type": "entry",
     "status": "generated",
     "content": "Dear [Tenant Name], ...",
     "statutory_citations": ["CA Civil Code § 1954.2"],
     "notice_period_days": 24,
     "effective_date": "2026-04-12",
     "disclaimer": "This notice was generated by an AI tool and should be reviewed by a legal professional before delivery.",
     "generated_at": "2026-04-10T14:22:00Z"
   }
   ```

5. **Database storage**:
   - Insert into compliance_notices table
   - status: "generated"
   - type: notice_type
   - content: generated notice text
   - jurisdiction_data: snapshot of rules used (JSONB)
   - Log to compliance_audit_log with action_type: "notice_generated"

6. **Error handling**:
   - 404 if property not found
   - 400 if jurisdiction not configured, or required context fields missing
   - 400 if notice_type invalid

7. **Update endpoints.md**
   - Document POST /api/compliance/notices/generate
