# /merge-to-main — Commit, Push, PR, and Merge to Main

Commit all uncommitted changes, push the current feature branch, create a pull request to main, and merge it.

Optional arguments: `$ARGUMENTS` (PR title override)

## Pre-flight Checks

### 1. Verify Branch State

```bash
git branch --show-current
git status --short
```

- STOP if on `main` — nothing to merge.

### 2. Commit Uncommitted Changes

If there are staged or unstaged changes (modified or untracked files):

```bash
git diff --name-only HEAD
git diff --cached --name-only
git status --short
```

Stage all relevant files and commit:
```bash
git add <specific changed files>
git commit -m "<descriptive message based on changes>"
```

Do NOT stage files that likely contain secrets (`.env`, `credentials.json`, etc.).
If there are no changes to commit, skip this step.

### 3. Verify Commits Ahead

```bash
git log main..HEAD --oneline
```

- STOP if no commits ahead of main — nothing to merge.

## Execution

### 4. Push Branch

```bash
git push -u origin $(git branch --show-current)
```

If push fails, report the error and STOP.

### 5. Create Pull Request

Determine the PR title:
- If `$ARGUMENTS` is not empty, use it as the title.
- Otherwise, derive from the branch name (e.g., `feature/P2-002-auto-scheduling-vendors` → `Feature: P2-002 Auto-scheduling Vendors`).

Generate a PR body from the commit log:

```bash
gh pr create \
  --title "<title>" \
  --base main \
  --body "$(cat <<'EOF'
## Summary

<auto-generated from commits>

## Commits

<list of commits from git log main..HEAD --oneline>

---
Generated by /merge-to-main
EOF
)"
```

Report the PR URL.

### 6. Merge the PR

```bash
gh pr merge --merge --delete-branch
```

If merge fails (e.g., CI checks, merge conflicts):
- Report the error
- Report the PR URL so the user can resolve manually
- STOP — do not force merge

### 7. Checkout Main

```bash
git checkout main
git pull origin main
```

### 8. Update Lifecycle (if applicable)

If `.claude/feature-lifecycle.json` exists and tracks the current feature:
- Update state to `"merged"`
- Update `updated_at`
- Commit the lifecycle update on main:
  ```bash
  git add .claude/feature-lifecycle.json
  git commit -m "Update lifecycle: <feature> merged to main"
  git push origin main
  ```

### 9. Report

- Branch pushed
- PR created (URL)
- PR merged
- Now on main
- Suggest next: run `/nextstep` to start the next feature
