# 220 — Notice Generator UI

## Objective
Build a multi-step wizard UI for creating, previewing, and sending legally-templated notices. Users select a property, choose a notice type, provide context, see a preview, and send. Includes disclaimer banners on every step.

## Context
Reference: `features/inprogress/P3-003-legal-compliance-engine/README.md`

The notice generator is a flagship feature of the compliance engine. It guides landlords through creating jurisdiction-specific notices without needing legal expertise.

## Implementation

1. **Create notice generator page** — `apps/web/app/(landlord)/compliance/notices/page.tsx`
   - Redirect to step 1 of the wizard

2. **Multi-step wizard** — `/compliance/notices/create`
   - Use React state or URL params to track current step
   - Steps: 1-property, 2-notice-type, 3-context, 4-preview, 5-send

3. **Step 1: Select Property**
   - Title: "Select Property"
   - DisclaimerBanner at top
   - List all user's properties with jurisdiction badge
   - Property cards show: name, address, jurisdiction (state + city)
   - Click property to advance to step 2
   - "Cancel" button returns to /compliance
   - Validation: Jurisdiction must be configured
     - If property has no jurisdiction: show "Configure Jurisdiction" message + link to task 222

4. **Step 2: Select Notice Type**
   - Title: "Notice Type"
   - DisclaimerBanner
   - Property name displayed
   - Radio button options:
     - "Entry Notice" — For landlord entry
     - "Lease Violation" — For tenant violation of lease
     - "Rent Increase" — For increasing rent
     - "Eviction Notice" — For eviction proceedings
   - Each option includes brief description of when to use
   - "Back" button → step 1, "Next" button → step 3
   - Validation: Notice type must be selected

5. **Step 3: Provide Context**
   - Title: "Notice Details"
   - DisclaimerBanner
   - Property name + notice type displayed
   - Form fields vary by notice type:

     **Entry Notice**:
     - Tenant name (text field)
     - Purpose of entry (dropdown: "Inspection", "Repair", "Other")
     - Proposed entry date (date picker)
     - Additional details (text area, optional)

     **Lease Violation**:
     - Tenant name (text field)
     - Violation description (text area)
     - Type of violation (dropdown: "Non-payment", "Property damage", "Nuisance", "Other")
     - Additional details (text area, optional)

     **Rent Increase**:
     - Tenant name (text field)
     - Current rent amount (number)
     - New rent amount or percentage increase (number + radio)
     - Effective date (date picker)
     - Reason (text area, optional)

     **Eviction Notice**:
     - Tenant name (text field)
     - Reason (dropdown: "Non-payment", "Lease violation", "No-cause termination")
     - Issue details (text area)
     - Cure deadline (date picker, optional, depends on reason)

   - "Back" button → step 2, "Generate" button → step 4
   - Validation: All required fields must be filled

6. **Step 4: Preview Generated Notice**
   - Title: "Preview Notice"
   - DisclaimerBanner
   - Generated notice displayed in formatted box (letterhead-style):
     - Formatted text with proper spacing
     - Statute citations highlighted
     - Notice period and effective date prominently shown
   - Metadata below:
     - Jurisdiction (state + city)
     - Notice period required (days)
     - Statute citations
     - Generated at timestamp
   - Buttons:
     - "Back to Edit" → returns to step 3
     - "Send Notice" → step 5
   - Edit notice text option (optional advanced feature):
     - Text area to manually edit generated content
     - Warning: "Editing generated content may remove required legal language"

7. **Step 5: Send Confirmation**
   - Title: "Send Notice"
   - DisclaimerBanner
   - Confirmation message: "Review the notice one more time before sending"
   - Notice preview (read-only)
   - Delivery method selector (optional):
     - Radio buttons: "Email", "Print (I'll deliver)", "Other"
   - Checkbox: "I have reviewed this notice with a legal professional" (optional but encouraged)
   - Warning box (yellow):
     - "This notice was generated by an AI tool. It should be reviewed by a qualified attorney before delivery."
   - Buttons:
     - "Back to Edit" → step 4
     - "Send Notice" → calls POST /api/compliance/notices/[id]/send

8. **After send**:
   - Success message: "Notice sent successfully"
   - Show notice details:
     - Sent date/time
     - Delivery method
     - Link to audit log
   - "View Notice" button → read-only view of sent notice
   - "Create Another Notice" button → restart wizard
   - "Back to Dashboard" button → /compliance

9. **Design patterns**:
   - Use multi-step form component (card-based with step indicator)
   - Show step number (e.g., "Step 1 of 5")
   - Progress bar at top
   - DisclaimerBanner on every step
   - Form validation with error messages
   - Loading state during notice generation (spinning icon)

10. **Error handling**:
   - Network error during generation: show error message + "Try again" button
   - Validation errors: highlight fields, show error messages
   - Property not found: redirect to /compliance
   - Jurisdiction missing: show helpful message with link to configure

11. **Update endpoints.md**
   - Document new pages: /compliance/notices/create, /compliance/notices/[id]
